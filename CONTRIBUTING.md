# Contributing to Laravel Search Barangay

First off, thank you for considering contributing to Laravel Search Barangay! It's people like you that make the open-source community such an amazing place.

## Code of Conduct

By participating in this project, you agree to abide by our code of conduct.

## How Can I Contribute?

### Reporting Bugs
- Check the [issue tracker](https://github.com/yahaaylabs/laravel-barangay-search/issues) to see if the bug has already been reported.
- If not, create a new issue, providing as much detail as possible, including steps to reproduce.

### Suggesting Enhancements
- Check the issue tracker to see if the enhancement has already been suggested.
- If not, create a new issue clearly describing the replacement or new feature.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Write your code and ensure it follows the [PSR-12](https://www.php-fig.org/psr/psr-12/) coding standard.
4. Write tests for your changes.
5. Ensure all tests pass by running `vendor/bin/pest`.
6. Submit a pull request to the `main` branch.

## Development Setup

1. Clone your fork.
2. Install dependencies:
   ```bash
   composer install
   ```
3. Run tests:
   ```bash
   composer test
   ```

## Coding Standards

We follow the PHP-FIG standards, specifically PSR-4 for autoloading and PSR-12 for coding style. Please ensure your contributions adhere to these standards.
